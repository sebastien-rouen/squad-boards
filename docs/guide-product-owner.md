# Guide Product Owner

> Suivez vos features, mesurez l'avancement PI et communiquez avec vos stakeholders.

---

## Ce que le board vous apporte

- Vision instantanee de l'avancement des **features** et **epics**
- Suivi des **objectifs PI** avec statut editable
- **Rapports** copiables pour les stakeholders (Slack, Confluence)
- **Sante du backlog** : tickets sans epic, sans estimation, sans priorite
- **Roadmap** avec allocation par equipe et velocity

## Vos ecrans quotidiens

### 1. Dashboard (`1`)
Vue d'ensemble : metriques sprint, cartes equipes, activite recente. Filtrez par equipe ou groupe dans la sidebar.

### 2. Roadmap (`5`)
- **Velocity breakdown** : points features vs buffer
- **Sante du backlog** : KPIs (sans epic, sans estimation, sans priorite)
- **Timeline features** : progression par feature avec barre
- **Allocation equipes** : qui travaille sur quoi

### 3. PI Planning (`4`)
- Onglet **Objectifs** : ajoutez, editez le statut (todo/inprog/done) directement
- Onglet **Features** : progression de chaque feature
- Onglet **Capacite** : membres, points, absences par equipe

### 4. Rapports (`6`)
- 7 sections : Sprint, Flow, Support, Roadmap, Equipes, PI, Complet
- 3 formats : Texte, **Slack** (preview dark), **Confluence** (preview table)
- Bouton copier par section

## Piloter les objectifs PI

1. Vue **PI Planning** (`4`) → onglet Objectifs
2. Ajoutez un objectif (texte + equipe)
3. Changez le statut inline via le selecteur
4. Le panneau info a droite resume l'avancement
