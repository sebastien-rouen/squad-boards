# Guide Project Manager

> Suivez l'avancement global, pilotez les risques et reportez aux parties prenantes.

---

## Vue d'ensemble

### Dashboard (`1`)
- **KPIs colores** : tickets, points, en cours, bloques - avec indicateurs visuels
- **Cartes equipes** : progression par equipe avec barre de completion
- **Activite recente** : derniers changements de statut

### Roadmap (`5`)
- **Velocity breakdown** : repartition features vs buffer
- **Sante du backlog** : tickets sans epic, sans estimation, priorite par defaut
- **Allocation equipes** : qui travaille sur quoi, quelle charge

## Piloter les risques

- **Alertes sprint** (vue Sprint) : retard, tickets bloques, WIP eleve
- **Faits marquants** (Parametres) : tracez incidents, gels, jalons
- **Panneau info** : vision temps reel des blocages et absences

## Rapports stakeholders

1. Vue **Rapports** (`6`)
2. Selectionnez le format : **Confluence** pour les wiki, **Slack** pour les channels
3. 7 sections disponibles : Sprint, Flow, Support, Roadmap, Equipes, PI, Complet
4. **Copiez** chaque section individuellement

## Suivi PI

- **PI Planning** (`4`) : objectifs avec statut, features, capacite par equipe
- **Config PI** (Parametres) : numero, sprints/PI, velocity cible
