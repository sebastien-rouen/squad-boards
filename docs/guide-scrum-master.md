# Guide Scrum Master

> Pilotez vos sprints, detectez les blocages et animez vos rituels depuis un seul outil.

---

## Votre journee type

### Le matin
1. Ouvrez la vue **Sprint** (`2`) - verifiez les alertes (bloques, WIP eleve, perimetre elargi)
2. Consultez le **panneau info** a droite : progression points, statuts, absences
3. Verifiez la **banniere support** : qui est au support cette semaine ?

### Pendant le sprint
- Utilisez les **quick filters** pour isoler les tickets bloques, non assignes, ou critiques
- Basculez entre les **3 modes** du board : colonnes, swimlanes (par assignee), liste
- Suivez le **burndown** et le **burnup** : courbe reelle vs ideale
- Surveillez le **WIP Age** : tickets en cours depuis trop longtemps (P85)

### En fin de sprint
- Consultez les **rapports** (`6`) pour preparer la review
- Copiez le rapport au format **Slack** ou **Confluence** en un clic
- Verifiez le **throughput** et le **cycle time** pour la retro

## Rituels

### Daily standup
- Vue Sprint, section **Activite du jour** : changements groupes par jour, transitions visibles
- Quick filter **bloques** pour les impediments a traiter
- Cliquez un ticket pour le detail complet (modal avec prev/next)

### Sprint Review
- Vue **Dashboard** (`1`) : metriques globales, cartes equipes, progression features
- Vue **Rapports** : section Sprint au format Slack pour partager

### Retrospective
- **Cycle Time scatter** : identifier les tickets qui ont traine
- **WIP Age** : detecter les patterns de blocage
- **Throughput** : tendance de debit sur le sprint
